# Product Requirements Document
# 사내 게시판 웹사이트

---

**문서 버전:** v1.0  
**작성일:** 2026-03-24
**프로젝트명:** 사내 게시판 시스템

---

## 1. 프로젝트 개요

본 프로젝트는 임직원들이 자유롭게 의견을 공유하고 소통할 수 있는 웹 기반 사내 게시판 시스템을 구축하는 것을 목표로 합니다. 직원들은 게시글을 작성하고, 댓글과 좋아요를 통해 상호작용하며, 필요에 따라 익명으로 의견을 표현할 수 있습니다.

---

## 2. 목표 및 배경

### 2.1 프로젝트 목표

- 직원 간 원활한 커뮤니케이션 채널 제공
- 익명 의견 표현을 통한 솔직한 피드백 문화 조성
- 직원 참여도 향상 및 조직 내 투명성 제고
- 사용하기 쉬운 인터페이스로 접근성 극대화

### 2.2 배경

기존의 이메일이나 메신저 기반 커뮤니케이션은 의견의 체계적인 관리와 공유가 어렵습니다. 사내 게시판을 통해 의견을 구조화하고, 인기 있는 주제를 쉽게 파악하며, 직원들의 참여를 독려할 수 있는 플랫폼이 필요합니다.

## 4. 사용자 스토리

### US-001: 게시글 작성
*As a* **직원**, *I want to* **게시판에 글을 작성** *so that* **나의 의견을 동료들과 공유할 수 있다**.

### US-002: 익명 작성
*As a* **직원**, *I want to* **익명으로 글을 작성** *so that* **신원 노출 없이 솔직한 의견을 낼 수 있다**.

### US-003: 게시글 조회 및 정렬
*As a* **직원**, *I want to* **게시글을 다양한 기준으로 정렬하여 조회** *so that* **관심 있는 주제나 인기 있는 글을 빠르게 찾을 수 있다**.

### US-004: 댓글 작성
*As a* **직원**, *I want to* **게시글에 댓글을 작성** *so that* **다른 의견에 대한 피드백을 제공할 수 있다**.

### US-005: 좋아요 표시
*As a* **직원**, *I want to* **게시글과 댓글에 좋아요를 표시** *so that* **공감하는 의견을 간편하게 표현할 수 있다**.

### US-006: 내 게시글 관리
*As a* **직원**, *I want to* **내가 작성한 게시글을 수정하거나 삭제** *so that* **잘못된 내용을 바로잡거나 불필요한 글을 제거할 수 있다**.

## 5. 데이터 모델

### 5.1 User (사용자)

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| id | String | Y | 사용자 고유 ID |
| email | String | Y | Google 이메일 |
| name | String | Y | 사용자 이름 |
| createdAt | DateTime | Y | 가입일시 |

### 5.2 Post (게시글)

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| id | String | Y | 게시글 고유 ID |
| title | String | Y | 제목 |
| content | Text | Y | 본문 |
| authorId | String | Y | 작성자 ID (FK) |
| isAnonymous | Boolean | Y | 익명 여부 |
| likesCount | Integer | Y | 좋아요 수 |
| commentsCount | Integer | Y | 댓글 수 |
| createdAt | DateTime | Y | 작성일시 |
| updatedAt | DateTime | N | 수정일시 |

### 5.3 Comment (댓글)

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| id | String | Y | 댓글 고유 ID |
| content | Text | Y | 댓글 내용 |
| postId | String | Y | 게시글 ID (FK) |
| authorId | String | Y | 작성자 ID (FK) |
| likesCount | Integer | Y | 좋아요 수 |
| createdAt | DateTime | Y | 작성일시 |

### 5.4 Like (좋아요)

| 필드명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| id | String | Y | 좋아요 고유 ID |
| userId | String | Y | 사용자 ID (FK) |
| postId | String | N | 게시글 ID (FK, 선택) |
| commentId | String | N | 댓글 ID (FK, 선택) |
| createdAt | DateTime | Y | 좋아요 누른 일시 |

**※ postId와 commentId 중 하나는 반드시 값이 있어야 합니다. userId+postId 또는 userId+commentId 조합은 고유해야 합니다 (1인당 1개 제약).**

## 6. 비기능 요구사항

### 6.1 성능

- 페이지 로딩 시간: 3초 이내
- 게시글 목록 페이지네이션: 20개 단위
- API 응답 시간: 평균 500ms 이하

### 6.2 보안

- HTTPS 통신 필수
- Google OAuth 2.0을 통한 안전한 인증
- 회사 도메인 계정만 접근 가능
- XSS, CSRF 등 웹 취약점 방어
- 익명 게시글의 실제 작성자 정보는 관리자만 확인 가능

### 6.3 사용성

- 직관적이고 사용하기 쉬운 UI/UX
- 반응형 디자인으로 모바일 환경 지원
- 주요 브라우저(Chrome, Safari, Firefox, Edge) 호환

### 6.4 확장성

- 향후 첨부파일 업로드 기능 추가 가능하도록 설계
- 카테고리/태그 기능 추가 가능
- 알림 기능 추가 가능

## 8. 화면 구성

### 8.1 로그인 페이지

- Google 로그인 버튼
- 서비스 소개 문구

### 8.2 게시글 목록 페이지 (메인)

- 상단: 네비게이션 바 (로고, 글쓰기 버튼, 사용자 프로필)
- 정렬 드롭다운 (최신순/댓글많은순/좋아요많은순)
- 게시글 목록 (카드 형식)
- 각 게시글: 제목, 작성자, 작성일시, 댓글 수, 좋아요 수 표시
- 페이지네이션

### 8.3 게시글 작성 페이지

- 제목 입력 필드
- 본문 입력 필드 (텍스트 에디터)
- 익명 작성 체크박스
- 등록 버튼, 취소 버튼

### 8.4 게시글 상세 페이지

- 게시글 제목, 작성자, 작성일시
- 게시글 본문
- 좋아요 버튼 및 좋아요 수
- 수정/삭제 버튼 (작성자에게만 표시)
- 댓글 목록
- 각 댓글: 작성자, 내용, 작성일시, 좋아요 버튼 및 좋아요 수
- 댓글 입력 폼
