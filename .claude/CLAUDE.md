## 모든 에이전트 공통 규칙

### 언어 및 톤
- 모든 코드 주석과 문서는 **한국어**로 작성
- 변수명, 함수명은 **영어**로 작성 (camelCase 사용)
- 사용자 친화적이고 직관적인 메시지 작성

### 코딩 스타일
- **TypeScript** 우선 사용
- **ESLint + Prettier** 설정 준수
- **함수형 컴포넌트** 및 **React Hooks** 사용
- **async/await** 패턴 사용 (Promise.then 금지)
- **Supabase 클라이언트** 표준 사용

### MCP 연동 규칙
- **Database Architect**: 모든 DB 작업은 Supabase MCP 활용
- **UI Designer**: 디자인 자산 생성 시 Monet MCP 우선 사용
- **QA Specialist**: 테스트 자동화는 Playwright MCP 필수 사용

### 보안 원칙
- 모든 API 요청에 인증 토큰 검증
- 사용자 입력값 검증 및 sanitization
- 익명 게시글 작성자 정보 보호

```
outputs/
├── management/        # 프로젝트 관리 문서 (Project Manager)
├── database/          # Supabase 스키마 및 설정 (Database Architect)
├── backend/           # API 및 서버 로직 (Backend Developer)
├── design/            # 디자인 시스템 및 자산 (UI Designer)
├── frontend/          # React 컴포넌트 및 페이지 (Frontend Developer)
├── testing/           # 테스트 코드 및 리포트 (QA Specialist)
├── review/            # 코드 리뷰 및 감사 (Final Reviewer)
└── documentation/     # 통합 문서
```

## 절대 금지 사항

❌ **하드코딩된 비밀키나 토큰을 코드에 직접 포함**
❌ **사용자 비밀번호를 평문으로 저장**
❌ **SQL Injection 취약점이 있는 쿼리 작성**
❌ **회사 도메인 외 이메일 계정 접근 허용**
❌ **익명 게시글 작성자 정보를 클라이언트에 노출**
❌ **CORS 설정을 '*'로 전체 허용**
❌ **에러 메시지에 민감한 시스템 정보 포함**
❌ **에이전트 간 협업 없이 독립적으로 작업 진행**
❌ **MCP 도구 사용 없이 해당 영역 작업 수행**

## 멀티 에이전트 협업 체크리스트

### Phase 1: 프로젝트 초기화 (Project Manager 주도)
- [ ] 전체 프로젝트 계획 수립
- [ ] 에이전트별 역할 및 책임 정의
- [ ] 협업 워크플로우 설정
- [ ] MCP 도구 연동 확인

### Phase 2: 설계 단계 (Database Architect + UI Designer)
- [ ] Supabase 스키마 설계 (Supabase MCP 활용)
- [ ] RLS 정책 및 실시간 구독 설정
- [ ] 디자인 시스템 구축 (Monet MCP 활용)
- [ ] UI/UX 가이드라인 작성

### Phase 3: 개발 단계 (Backend + Frontend Developer)
- [ ] Supabase 연동 API 개발
- [ ] 인증 시스템 구현
- [ ] React 컴포넌트 개발
- [ ] 실시간 기능 구현

### Phase 4: 품질 검증 (QA Specialist)
- [ ] E2E 테스트 구현 (Playwright MCP 활용)
- [ ] 단위 테스트 및 통합 테스트
- [ ] 성능 및 접근성 테스트
- [ ] 버그 리포트 및 개선 제안

### Phase 5: 최종 검수 (Final Reviewer)
- [ ] 전체 코드 품질 검토
- [ ] 보안 취약점 분석
- [ ] 배포 준비 상태 점검
- [ ] 최종 승인 및 배포 권고
